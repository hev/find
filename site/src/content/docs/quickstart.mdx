This walks through adding search to an existing Astro 5 docs site whose content
lives in a collection (for example `src/content/docs`). It takes about five
minutes. The keyword path works immediately; the agentic path turns on once you
add an API key.

## Prerequisites

- **Astro 5** with at least one [content collection](https://docs.astro.build/en/guides/content-collections/).
- A **server or hybrid adapter** (Node, Cloudflare, Vercel, …). The search
  endpoint renders on demand, so a fully static build can't serve it.
- Optional: an **`ANTHROPIC_API_KEY`** for the agentic search loop.

### 1. Install

Once published, install from npm:

```sh
pnpm add @hev/ask
```

Until then, consume the latest version straight from the package subdirectory
on GitHub:

```sh
pnpm add "git+ssh://git@github.com/hev/ask.git#main&path:/packages/ui"
```

### 2. Register the integration

```js
// astro.config.mjs
import { defineConfig } from "astro/config";
import hevAsk from "@hev/ask";

export default defineConfig({
  integrations: [
    hevAsk({
      collections: ["docs"],   // your content collection name(s)
      basePath: "/docs/",      // slug → URL prefix: basePath + slug
    }),
  ],
});
```

`collections` is the one option you must set. Everything else has a default —
see the [configuration reference](/docs/api/configuration).

### 3. Add a server adapter

The `/api/ask` route is rendered on demand. Add whichever adapter matches your
host; this site uses Cloudflare:

```js
// astro.config.mjs
import cloudflare from "@astrojs/cloudflare";

export default defineConfig({
  adapter: cloudflare({ platformProxy: { enabled: true } }),
  // ...integrations as above
});
```

Existing pages stay prerendered. Only `/api/ask` runs as a function.

### 4. Render the overlay

Add the component once, somewhere global like your base layout, and give
readers a way to open it. **Any element with `data-hev-ask-open` opens the
palette**, and `⌘K` is bound automatically.

```astro
---
// src/layouts/Base.astro
import SearchOverlay from "@hev/ask/components/SearchOverlay.astro";
---
<button type="button" data-hev-ask-open>
  Search <kbd>⌘K</kbd>
</button>

<slot />

<SearchOverlay />
```

At this point keyword search works. Run `astro dev`, press `⌘K`, and type.

### 5. Build the knowledge graph

The knowledge graph is an offline-built JSON file committed to your repo. It
gives the agentic loop domain context, ranks keyword results, supplies the
glossary (so `k8s` finds `kubernetes`), and holds the questions the overlay
suggests. Only the distillation is model-authored; everything else is derived in
code. There are two ways to produce that distillation:

**Recommended — the Claude Code skill.** If you work in
[Claude Code](https://claude.com/claude-code), install the bundled skill and ask
for it. It builds the graph inside your subscription — **no API key, no token
spend on your own key**:

```text
You: build the hev ask knowledge graph

Claude runs:
  hev-ask-kg corpus            # emits the sections to distil
  …writes context/glossary/summaries/suggestions…
  hev-ask-kg assemble          # writes .hev-ask/kg.json (deterministically)
```

**Fallback — the CLI.** For CI or outside Claude Code, one API call does the same
distillation:

```sh
export ANTHROPIC_API_KEY=sk-ant-...
pnpm exec hev-ask-kg build      # writes .hev-ask/kg.json
```

Then verify and commit, however you built it:

```sh
pnpm exec hev-ask-kg verify     # builds the site, checks every anchor resolves
git add .hev-ask/kg.json
```

Both paths are **hash-gated** — they do no model work when content is unchanged,
so they're cheap to re-run. The integration also runs `build` automatically
during `astro build` when a key is present. See the
[CLI reference](/docs/api/cli) for every command.

<Callout label="no key, no problem">
Skip step 5 entirely and hev ask still works in keyword mode. Without a graph
the agentic path falls back to keyword results, keyword ranking falls back to
raw token overlap, and the overlay simply shows no suggestions. You can build
the graph later.
</Callout>

## Turn on agentic search

Set `ANTHROPIC_API_KEY` in the **server environment** where `/api/ask` runs
(your host's secrets, or `.env` in local dev). With a key present, pressing
Enter in the overlay runs the [agentic loop](/docs/concepts#the-agentic-search-loop);
without one, Enter returns keyword results.

## Verify it works

- **Keyword:** type a single word from a heading; the top result should
  deep-link to that section (`/docs/page#heading`).
- **Agentic:** type a multi-word question (or click a suggested one) and press
  Enter; the footer shows the sub-queries the model ran, and a grounded answer
  streams in with inline deep links.
- **Anchors:** `hev-ask-kg verify` exits non-zero if any chunk anchor doesn't
  exist in the built HTML — wire it into CI.
